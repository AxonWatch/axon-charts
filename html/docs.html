Introduction

LLM-first financial charting library for modern web applications.

Try It Live

What is Axon Charts?

Axon Charts is a canvas-based financial charting library built from the ground up for AI agents and developers. It provides:

  • LLM-First Design: Every feature is accessible via simple, predictable APIs that AI agents can discover and use
  • Lightning Fast: Handles 5000+ candles at 60fps with optimized draw calls
  • 94 Configuration Options: Customize every aspect of appearance and behavior
  • TypeScript Support: Full type definitions for excellent developer experience
  • Production Ready: Volume sub-panes, streaming support, market headers, and more

Key Features

Feature Description
Real-time Updates WebSocket support with updateLastBar() and updateLastBarFast() methods
Volume Sub-pane Display volume histogram below main chart with auto precision detection
Crosshair Modes Normal, magnet (snap-to-bar), and disabled modes
Context Export getContext() returns structured JSON for LLM analysis
94 Options Layout, grid, scales, colors, behavior, and more

Browser Support

Works on all modern browsers that support ES6 and Canvas API:

  • Chrome 90+
  • Firefox 88+
  • Safari 14+
  • Edge 90+

Quick Start

Try It Live

Basic Chart

// Import the library
import { createChart } from 'axon-charts';

// Create a chart
const chart = createChart('#container', {
  layout: {
    background: '#1a1a1a',
    textColor: '#aaa',
  },
  grid: {
    show: true,
    vertLines: { color: '#2a2a2a' },
    horzLines: { color: '#2a2a2a' },
  },
});

// Load data
chart.setData([
  { time: 1609459200000, open: 100, high: 105, low: 98, close: 103 },
  { time: 1609459260000, open: 103, high: 107, low: 102, close: 105 },
  // ... more data
]);

Real-time Updates

// Update the latest candle
chart.updateLastBar({
  time: Date.now(),
  open: 105,
  high: 108,
  low: 104,
  close: 107,
});

Get Chart State

// Export chart context for LLM analysis
const ctx = chart.getContext();
console.log(ctx.visibleBars);
console.log(ctx.viewport);
console.log(ctx.priceRange);

Layout

Chart dimensions, colors, fonts, and padding — 10 options.

Try It Live

Width & Height

Set to 'auto' (reads container size) or a fixed pixel value. Auto is recommended for responsive layouts.

layout: {
  width: 'auto',    // reads container.clientWidth
  height: 'auto',   // reads container.clientHeight
  background: '#1a1a1a',
  textColor: '#ffffff',
  fontSize: 12,
  fontFamily: 'system-ui'
}

Padding

Controls chart margins. Maps directly to internal state variables.

layout: {
  padding: {
    top: 40,     // → state.topMargin
    right: 60,   // → state.axisWidth
    bottom: 35,  // → state.bottomMargin
    left: 10     // reserved
  }
}

Dynamic Updates

chart.setOptions({ layout: { background: '#000000', textColor: '#ffffff' } });
chart.setOptions({ layout: { padding: { top: 20, right: 40 } } });

Price Scale

Y-axis behavior and formatting — 15 options.

Try It Live

Scale Mode

Linear is standard for most charts. Logarithmic is useful for assets with large price ranges.

priceScale: {
  mode: 'linear',           // 'linear' | 'logarithmic' | 'percentage'
  scaleMargins: {
    top: 0.1,               // 10% padding above max price
    bottom: 0.1             // 10% padding below min price
  }
}

Percentage Mode

Normalizes all prices relative to the close price of the leftmost visible bar. The reference updates dynamically as you scroll or zoom. Display-only — internal data remains in original prices. Switch back to linear mode to restore numeric axis labels.

Axis labels, current price line, and crosshair price label use smart adaptive formatting:

Change Display Range
+2.45+2.45%Small changes (2 decimal places)
+245.8+245.8%Medium changes (1 decimal place)
+8,450+8,450%Large changes (thousands separator, 0 decimals)
+80,500+805.0×Extreme moves ≥10,000% (× multiplier)

× multiplier activates for changes of 100× or more (e.g. Bitcoin from $100 to $80k+). Formatting uses Intl.NumberFormat for proper locale-aware thousands separators.

Price Format

priceScale: {
  priceFormat: {
    type: 'price',           // 'price' | 'volume' | 'percent'
    precision: 2,            // decimal places
    minMove: 0.01            // tick increment (derives precision if not set)
  }
}

Current Price Line

Dashed horizontal line at the latest close price with a label on the Y-axis. Colors cascade: if not set explicitly, they follow series colors and layout.textColor.

priceScale: {
  currentPrice: {
    show: true,              // master toggle for line + label
    showLine: true,          // show/hide only the dashed line
    showCountdown: false,    // candle-close countdown timer (default: false)
    countdownColor: 'rgba(255, 255, 255, 0.8)',
    upColor: null,           // bull line color — falls back to series.upColor
    downColor: null,         // bear line color — falls back to series.downColor
    lineStyle: 'dashed',     // 'dashed' | 'solid'
    textColor: null          // label text — falls back to layout.textColor
  }
}
  • upColor / downColor — when null, follows candle colors. Set explicitly to decouple from candles.
  • textColor — when null, follows layout.textColor. Essential for light backgrounds where white text would be invisible.
  • showLine — hides only the dashed horizontal line while keeping the price label visible.
  • countdownColor — when null, falls back to textColor, then layout.textColor.
  • lineStyle — dashed (default) or solid.

The label background uses the line color at 80% opacity. Line style supports both dashed (default) and solid.

chart.setOptions({ priceScale: { currentPrice: { textColor: '#000000', lineStyle: 'solid' } } });

Reverse

Inverts the price axis so high prices appear at the bottom:

chart.priceScale().setReverse(true);
chart.execute({ type: 'setReverse', reverse: true });

Time Scale

X-axis behavior, timezone, and formatting — 11 options.

Try It Live

Visibility & Format

timeScale: {
  visible: true,
  timeVisible: true,         // show time on axis (false = date only)
  secondsVisible: false,     // HH:mm:ss format
  showFullDate: true,        // crosshair: "Fri 03 Jan'26 17:55"
  showDayOfWeek: true,       // show "Mon", "Tue" prefix in crosshair
  dateFormat: 'MMM dd, yyyy' // tokens: yyyy, yy, MMM, MM, dd
}

Timezone

Controls how timestamps are displayed on the axis and crosshair. By default, uses the browser's local timezone.

timeScale: {
  timezone: undefined         // browser local (default)
  // timezone: 'UTC'          // universal time
  // timezone: 'America/New_York'   // any IANA name
}

Uses Intl.DateTimeFormat — handles daylight saving automatically. The crosshair tooltip and all axis labels respect the configured timezone.

Bar Spacing

timeScale: {
  barSpacing: 11,            // base bar width in pixels
  minBarSpacing: 4,          // zoom-in boundary
  maxBarSpacing: 1000        // zoom-out boundary
}
chart.timeScale().setBarSpacing(20);

Right Offset

Gap in pixels between the last bar and the right edge of the chart area:

timeScale: {
  rightOffset: 80
}
chart.setOptions({ timeScale: { rightOffset: 40 } });
chart.setOptions({ timeScale: { timezone: 'UTC', dateFormat: 'yyyy-MM-dd' } });  // ISO-style

Menu

Right-click context menu — 2 options.

Try It Live

Master Toggle

Enable or disable the right-click context menu entirely.

menu: {
  enabled: true          // true = show, false = hide
}
chart.setOptions({ menu: { enabled: false } });

Custom Item Order & Filtering

Control which items appear in the menu and in what order. Set to null or omit to show all default items. Provide an array of item IDs to filter and reorder.

menu: {
  items: ['copy', 'save', 'grid', 'volume', 'fullscreen']     // filtered list
}

menu: {
  items: null                                                   // defaults: all items
}
chart.setOptions({ menu: { items: ['fullscreen', 'reset-price', 'grid'] } });

Dividers / Separators

The default order includes three dividers that group items logically:

Divider ID Position (default order) Separates
divider1After saveActions (Copy, Save) from toggles
divider2After watermarkToggles from remaining actions (Fit Content, Reset Price, Reverse)
divider3After reverseActions from Fullscreen

Any ID starting with divider creates a divider — divider1, divider2, divider, divider-anything. You can add as many as you want anywhere in a custom items array. Dividers do not auto-insert in custom arrays — they only appear where you explicitly place them.

menu: {
  items: ['copy', 'save', 'divider', 'grid', 'volume', 'divider2', 'fullscreen']
}

Toggle items show their state via bold (enabled) vs normal (disabled) weight.

Data

Data management and limits — 2 options.

Try It Live

Controls how the chart manages its internal data buffer — useful for long-running streaming sessions.

data: {
  maxBars: 5000,       // max candles kept in memory (1-1,000,000)
  autoCleanup: true    // auto-trim old bars when limit is exceeded
}
chart.setOptions({ data: { maxBars: 10000 } });

maxBars

Maximum number of bars stored in memory. Older bars are trimmed when this limit is exceeded. Default is 5000. Higher values consume more memory but allow further panning into history.

autoCleanup

When true (default), the chart automatically removes bars from the beginning of the dataset when maxBars is exceeded. Set to false if you manage trimming manually.

Context (LLM Exposure)

Control what AI agents can discover and read — 3 options.

Try It Live

These options control the AI agent integration layer. Three settings are available:

exposeData

Controls what getContext() returns. When false (default), only viewport metadata is returned — visible bars, latest bar, and sub-pane data are excluded. When true, full trading data is available to agents.

context: {
  exposeData: false    // metadata only — safe default
  // exposeData: true  // full OHLCV data exposure
}

Runtime-toggleable via chart.setOptions({ context: { exposeData: true } }).

discoverable

Controls whether the chart registers in the global window.__AXON_CHARTS__ registry. When true (default), AI agents with the Axon Charts skill can discover and interact with the chart. Set to false for stealth mode.

context: {
  discoverable: true   // visible to AI agents — default
  // discoverable: false // stealth mode
}

id

Optional identifier for the chart in the AI agent registry. Used to target specific charts in multi-chart environments.

  • If set explicitly: used as-is (e.g. 'Bitcoin-1m')
  • If not set: auto-generated opaque token (e.g. ax-uhq8fz) — no pair inference, no stale associations

Opaque IDs prevent stale instrument associations. Agents must read state.market in getContext() to determine the current instrument.

The ID is used as both the registry key and the data-axon-charts-id DOM attribute on the container element.

// Create with explicit ID
const chart = createChart('#container', {
  context: { id: 'Bitcoin-1m' },
  // ... other options
});

// Agent discovery in multi-chart page
window.__AXON_CHARTS__.charts['Bitcoin-1m'].setOptions({
  context: { exposeData: true }
});

Agent Discovery Flow

  1. Agent checks window.__AXON_CHARTS__ — sees chart ID and version
  2. Agent calls chart.getContext() — gets metadata (exposeData: false by default)
  3. Agent decides to request data access via chart.setOptions({ context: { exposeData: true } })
  4. Full context with bars, latest bar, and sub-panes is now available for analysis

Documentation / API Reference

Events

React to user interactions and chart state changes.

Crosshair Move

Fired when the user moves the cursor over the chart. Returns the bar under cursor, price, and timestamp.

chart.onCrosshairMove(({ bar, price, time }) => {
  console.log(`Price: $${price} at ${new Date(time)}`);
  console.log(`OHLC: O=${bar.open} H=${bar.high} L=${bar.low} C=${bar.close}`);
});

Bar Click

Fired when the user clicks on a specific bar/candle.

chart.onBarClick(({ bar, index }) => {
  console.log(`Clicked bar #${index}:`, bar);
});

Visible Range Change

Fired when the viewport changes (pan, zoom, resize). Returns the visible time range.

chart.onVisibleRangeChange(({ from, to }) => {
  console.log(`Viewing: ${new Date(from)} to ${new Date(to)}`);
});

Scroll Lock Change

Fired when auto-scroll locks/unlocks as the user pans through history.

chart.onScrollLockChange((locked) => {
  console.log(locked ? 'Exploring history' : 'Watching live');
});

Event Reference

Event Callback Parameter Description
onCrosshairMove { bar, price, time } Cursor position changed
onBarClick { bar, index } Bar/candle was clicked
onVisibleRangeChange { from, to, fromIndex, toIndex } Viewport changed via pan/zoom/resize
onScrollLockChange locked: boolean Auto-scroll state changed

ChartOptions

The main configuration object. All fields are optional — defaults fill in the rest.

interface ChartOptions {
  layout?: {
    width?: number | 'auto';      // default: 'auto'
    height?: number | 'auto';     // default: 'auto'
    background?: string;          // default: '#1a1a1a'
    textColor?: string;           // default: '#ffffff'
    fontSize?: number;            // default: 12
    fontFamily?: string;          // default: 'system-ui'
    borderVisible?: boolean;      // default: false
    padding?: {
      top?: number;               // default: 40
      right?: number;             // default: 60
      bottom?: number;            // default: 35
      left?: number;              // default: 10
    };
  };
  attribution?: {
    show?: boolean;               // default: true (Axon.Watch attribution at bottom-left)
  };
  grid?: {
    show?: boolean;               // default: true
    vertLines?: { show?, color?, width? };
    horzLines?: { show?, color?, width? };
  };
  series?: {
    type?: 'candlestick' | 'line' | 'area' | 'bar' | 'heiken-ashi' | 'hollow';  // default: 'candlestick'
    upColor?: string;             // default: '#10B981'
    downColor?: string;           // default: '#E11D48'
    lineColor?: string;           // default: '#1E90FF' (line/area only)
    showMarkers?: boolean;        // default: false (dot at every close)
    showLatestPriceMarker?: boolean;  // default: true (highlighted latest dot)
    showLatestPriceAnimation?: boolean; // default: true (pulse on price change)
  };
  priceScale?: {
    mode?: 'linear' | 'logarithmic' | 'percentage';  // default: 'linear'
    scaleMargins?: { top?, bottom? };
    priceFormat?: { type?, precision?, minMove? };
    currentPrice?: {
      show?: boolean;                    // default: true
      showLine?: boolean;                // default: true

      countdownColor?: string;          // default: 'rgba(255, 255, 255, 0.8)'

      downColor?: string;               // default: null (follows series.downColor)
      lineStyle?: 'dashed' | 'solid';   // default: 'dashed'
      textColor?: string;               // default: null (follows layout.textColor)
    };
    reverse?: boolean;            // default: false
  };
  timeScale?: {
    visible?: boolean;            // default: true
    timeVisible?: boolean;        // default: true
    secondsVisible?: boolean;     // default: false
    showFullDate?: boolean;       // default: true
    showDayOfWeek?: boolean;      // default: true
    dateFormat?: string;          // default: 'MMM dd, yyyy'
    timezone?: string;            // default: undefined (browser local)
    rightOffset?: number;         // default: 80
    barSpacing?: number;          // default: 11
    minBarSpacing?: number;       // default: 4
    maxBarSpacing?: number;       // default: 1000
  };
  crosshair?: {
    mode?: 'normal' | 'magnet' | 'none';  // default: 'magnet'
    showLabels?: boolean;         // default: true
    showTooltip?: boolean;        // default: true
    vertLine?: { color?, width?, style? };
    horzLine?: { color?, width?, style? };
  };
  menu?: {
    enabled?: boolean;            // default: true
    items?: string[] | null;      // default: null (all items, auto-dividers)
  };
  behavior?: {

Architecture

The library uses a three-layer canvas architecture for maximum performance:

Layer Canvas Renders
Background bgCanvas Fill, grid lines, price/time axes, watermark, volume sub-pane, percentage reference line
Main mainCanvas Candles (offscreen-buffered) + current price line (atomic render pass)
Overlay overlayCanvas Crosshair, tooltip, market header

Offscreen Buffer

The candle buffer is rendered at identity scale (1:1 pixel ratio) into an offscreen canvas. Only the visible portion is composited to the display canvas every frame. When a single candle updates (tick stream), only updateLastCandleInBuffer() is called — no full re-render.

Optimization Tips

  • Streaming: Use updateLastBarFast() for tick-by-tick updates (10-1000/sec). Call chart.render() every 20 ticks to refresh axes/grid.
  • Data Limits: Set data.maxBars to the minimum required for your viewport. 500 bars is sufficient for most charts.
  • Sub-Panes: Only enable volume/indicators when needed — each sub-pane adds render overhead.
  • Overlays: Overlay indicators (SMA, EMA, BB, etc.) re-compute every frame. For large datasets, pre-compute values and inject via SubPane.setData().
  • Persistence: saveState() captures drawings + overlays. Use migrateSnapshot() to upgrade old snapshots before loadState().
  • Resize: The library uses ResizeObserver with a re-entrancy guard — avoid manual resize calls when using auto-sizing containers.
  • Canvas Export: toDataURL() is the most expensive operation — debounce to every 500ms.
